# Tuist Project Generation

The `Client.xcodeproj` is **generated by Tuist** and enumerates source files explicitly. A new Swift file added to disk is NOT compiled until the project is regenerated.

## When to Run

Run `bash tuist-setup.sh` from the workspace root whenever you:

- Create a new Swift file anywhere under `firefox-ios/`
- Create a new asset catalogue or resource
- Add a new directory that should be included in a target

## Source Globs per Target

| Target | Glob |
| --- | --- |
| `Client` (app) | `Client/**/*.{swift,h,m}` |
| `Ecosia` (framework) | `Ecosia/**/*.{h,swift}` |
| `EcosiaTests` | defined in `Targets+Tests.swift` |

New files matching these globs are automatically picked up **after** `tuist-setup.sh` is run. No manual edits to `.xcodeproj` are needed.

## Verifying a New File Is Registered

```sh
grep "MyNewFile.swift" firefox-ios/Client.xcodeproj/project.pbxproj
```

It should appear twice: once as a `PBXFileReference` and once in a `PBXBuildFile` (Sources phase).

## Why Incremental Builds Can Silently Hide This

If a new file's symbols happen to be in DerivedData from a prior build, incremental builds will succeed. **Always verify with a clean build** after adding new files.

## Tuist Configuration Location

- `firefox-ios/Tuist/ProjectDescriptionHelpers/Targets+Frameworks.swift` — Ecosia framework target
- `firefox-ios/Tuist/ProjectDescriptionHelpers/Targets+Client.swift` — Client app target
- `firefox-ios/Tuist/ProjectDescriptionHelpers/Targets+Tests.swift` — test targets
